00014 /**
00015 * @addtogroup grp_linalg
00016 *
00017 * @brief
00018 * In linear algebra, the singular value decomposition (SVD) is a factorization of
00019 * a real or complex matrix, with many useful applications in signal processing and
00020 * statistics.
00021 *
00022 * Let \f$A\f$ be a \f$mxn\f$ matrix, where \f$m \ge n\f$. Then \f$A\f$ can be
00023 * decomposed as follows:
00024 * \f[
00025 *     A = U \Sigma V^T,
00026 * \f]
00027 * where \f$U\f$ is a \f$m \times n\f$ orthonormal matrix,
00028 * \f$\Sigma\f$ is a \f$n \times n\f$ diagonal matrix, and \f$V\f$ is an
00029 * \f$n \times n\f$ orthonormal matrix. The diagonal elements of \f$\Sigma\f$ are
00030 * called the \emph{singular values}.
00031 *
00032 *
00033 * @input
00034 *   The input table can be two forms:
00035         - Dense matrix representation
00036             The table contains a 'row_id' column that identifies each row.
00037             Further, the other columns are assumed to be the data for the matrix
00038             represented in two possible forms, illustrated in a the 2x2 matrix example below:
00039             -# <pre>
00040                             row_id     col1     col2
00041                 row1         1           1         0
00042                 row2         2           0         1
00043             </pre>
00044             -# <pre>
00045                             row_id     row_vec
00046                     row1        1       {1, 0}
00047                     row2        2       {0, 1}
00048             </pre>
00049         - Sparse matrix representation
00050             An example representation is given below:
00051             <pre>
00052                        row_id    col_id         value
00053             row1        1             1           1
00054             row2        2             2           1
00055             </pre>
00056             The 'row_id' represents the row number, 'col_id' represents the column
00057             id and the 'value' represents the matrix value at ['row_id', 'col_id']
00058 
00059 * @output
00060 *   The output and summary table
00061 *    -------------------------------------------------------------------------
00062 *                                OUTPUT TABLE
00063 *    -------------------------------------------------------------------------
00064 *    Ouptut for eigen vectors/values will be in the following 3 tables:
00065 *        - Left singular matrix: Table named <output_table_prefix>_left (e.g. ‘netflix_u’)
00066 *        - Right singular matrix: Table named <output_table_prefix>_right (e.g. ‘netflix_v’)
00067 *        - Singular values: Table named <output_table_prefix>_s (e.g. ‘netflix_s’)
00068 *    The singular vector tables are of the format:
00069 \code
00070         row_id              INTEGER,   -- ID corresponding to each eigen value (in decreasing order)
00071         row_vec             FLOAT8[]   -- Singular vector elements for this row_id
00072 \endcode                                          Each array is of size k
00073 *    The singular values table is in a sparse table format:
00074 \code
00075         row_id              INTEGER,   -- i for ith eigen value
00076         col_id              INTEGER,   -- i for ith eigen value (same as row_id)
00077         value               FLOAT8     -- Eigen Value
00078 \endcode
00079 
00080 *    -------------------------------------------------------------------------
00081 *                        RESULT SUMMARY TABLE
00082 *    -------------------------------------------------------------------------
00083 *    Result summary table will be in the following format:
00084 \code
00085 *        rows_used           INTEGER,    -- Number of rows used for SVD calculation
00086 *        exec_time           FLOAT8,     -- Total time for executing SVD
00087 *        iter                INTEGER,    -- Total number of iterations run
00088 *        recon_error         FLOAT8      -- Total quality score (i.e. approximation quality) for this set of
00089 *                                                orthonormal basis
00090          relative_recon_error FLOAT8     -- relative quality score
00091 \endcode
00092 
00093 In the result summary table, the reconstruction error is computed as \f$ \sqrt{mean((X - USV^T)_{ij}^2)} \f$, where the average is over all elements of the matrices. The relative reconstruction error is then computed as ratio of the reconstruction error and \f$ \sqrt{mean(X_{ij}^2)} \f$.
00094

00194 * @par Technical Background
00195 * In linear algebra, the singular value decomposition (SVD) is a factorization of
00196 * a real or complex matrix, with many useful applications in signal processing and
00197 * statistics.
00198 *
00199 * Let \f$A\f$ be a \f$m \times n\f$ matrix, where \f$m \ge n\f$. Then \f$A\f$ can be
00200 * decomposed as follows:
00201 * \f[
00202 *     A = U \Sigma V^T,
00203 * \f]
00204 * where \f$U\f$ is a \f$m \times  n\f$ orthonormal matrix,
00205 * \f$\Sigma\f$ is a \f$n \times n\f$ diagonal matrix, and \f$V\f$ is an
00206 * \f$n \times  n\f$ orthonormal matrix. The diagonal elements of \f$\Sigma\f$ are
00207 * called the \emph{singular values}.
00208 *
00209 * It is possible to formulate the problem of computing the singular triplets
00210 * (\f$\sigma_i, u_i, v_i\f$) of \f$A\f$ as an eigenvalue problem involving a Hermitian
00211 * matrix related to \f$A\f$. There are two possible ways of achieving this:
00212 *
00213 * -# With the cross product matrix, \f$A^TA\f$ and \f$AA^T\f$
00214 * -# With the cyclic matrix
00215 *       \f[
00216 *           H(A) =
00217 *       \begin{bmatrix}
00218 *       0   & A\\
00219 *       A^* & 0
00220 *       \end{bmatrix}
00221 *       \f]
00222 *
00223 * The singular values are the nonnegative square roots of the eigenvalues of the
00224 * cross product matrix. This approach may imply a severe loss of accuracy in the
00225 * smallest singular values. The cyclic matrix approach is an alternative that
00226 * avoids this problem, but at the expense of significantly increasing the cost of
00227 * the computation.
00228 *
00229 * Computing the cross product matrix explicitly is not recommended, especially in
00230 * the case of sparse A. Bidiagonalization was proposed by Golub and Kahan
00231 * [citation?] as a way of tridiagonalizing the cross product matrix without
00232 * forming it explicitly.
00233 *
00234 * Consider the following decomposition
00235 * \f[    A = P B Q^T, \f]
00236 * where \f$P\f$ and \f$Q\f$ are unitary matrices and \f$B\f$ is an \f$m \times n\f$
00237 * upper bidiagonal matrix. Then the tridiagonal matrix \f$B*B\f$ is unitarily
00238 * similar to \f$A*A\f$. Additionally, specific methods exist that compute the
00239 * singular values of \f$B\f$ without forming \f$B*B\f$. Therefore, after computing the SVD of B,
00240 * \f[
00241 *     B = X\Sigma Y^T,
00242 * \f]
00243 * it only remains to compute the SVD of the original matrix with \f$U = PX\f$ and \f$V = QY\f$.
00244 *
00245 **/
