Release Integrity

You must verify the integrity of the downloaded files. We provide OpenPGP signatures for every release file. This signature should be matched against the KEYS file which contains the OpenPGP keys of MADlib's Release Managers. We also provide MD5 and SHA-512 checksums for every release file. After you download the file, you should calculate a checksum for your download, and make sure it is the same as ours.

  • Future Releases

    Official releases are created when the community feels that the product has progressed sufficiently and is at a stable state. Watch the user mailing list for latest release announcements and the developer mailing list for ongoing discussions related to the current release status. As always we welcome new members to the community.

    • User Mailing List
    • Devel Mailing List

  • Backward Compatibility of Releases

    The MADlib project uses Semantic Versioning. All versions, starting with MADlib v1.0, will be API compliant and we will supply mechanisms for upgrading from prior versions to the current active version until the release of MADlib v2.0 which may have different API. In many cases this may mean that a particular function may become deprecated, yet still remain callable, until a later major version is released.
